## Обзор проекта

**BizFlow** — централизованная система учёта бизнеса на базе PHP 8.4 и Yii3.

### Назначение

Система предоставляет единый, достоверный контур учёта для:
- Товаров и остатков
- Движения товаров
- Сделок и клиентов
- Точек продаж
- Выручки и аналитики
- Документов по сделкам
- Интеграций с внешними системами

### Технологический стек

| Компонент | Технология |
|-----------|------------|
| Язык | PHP 8.4 |
| Фреймворк | Yii3 |
| База данных | MySQL 8 |
| Кэширование | Redis 7 |
| Контейнеризация | Docker + Docker Compose |
| Web-сервер | Caddy (в контейнере) |

### Архитектура

Проект следует принципам **DDD (Domain-Driven Design)**, **SOLID**, **DRY**, **KISS**.

#### Структура слоёв

1. **Domain** — Entities, Value Objects, Aggregates, Domain Services, Domain Events
2. **Application** — Use Cases (1 сценарий = 1 класс), DTO, Interfaces (ports)
3. **Infrastructure** — DB (MySQL), Redis, API интеграции, реализации репозиториев
4. **Presentation** — REST API

#### Структура каталогов

```
├── config/           # Конфигурации приложения (common, web, console)
├── src/              # Исходный код приложения
│   ├── Console/      # Команды CLI
│   ├── Database/     # Миграции БД
│   ├── Shared/       # Общие компоненты
│   └── Web/          # Веб-компоненты (контроллеры, действия)
├── tests/            # Тесты (Unit, Functional, Web, Console)
├── public/           # Публичная директория (index.php, assets)
├── docker/           # Docker-конфигурации (dev, test, prod)
├── docs/             # Документация
└── runtime/          # Временные файлы (кэш, логи)
```

## Сборка и запуск

### Команды Makefile

#### Development

| Команда | Описание |
|---------|----------|
| `make build` | Сборка Docker-образов для разработки |
| `make up` | Запуск окружения разработки |
| `make down` | Остановка окружения |
| `make stop` | Остановка контейнеров |
| `make clear` | Удаление контейнеров и volumes |
| `make shell` | Вход в bash-оболочку контейнера |

#### Инструменты

| Команда | Описание |
|---------|----------|
| `make yii <command>` | Выполнение команды Yii CLI |

### Основные команды

```bash
# Запуск окружения
make up

# Выполнение миграций
make yii migrate

# Запуск тестов
make test

# Проверка кода
make psalm
make cs-fix
```

### Конфигурация окружения

Скопируйте `.env.example` в `.env` и настройте переменные:

```bash
cp .env.example .env
```

Ключевые переменные:
- `APP_ENV` — окружение (dev/test/prod)
- `APP_DEBUG` — режим отладки
- `DB_DSN`, `DB_USER`, `DB_PASSWORD` — подключение к MySQL
- `REDIS_DSN` — подключение к Redis

## Правила разработки

### Стиль кодирования

- **Форматирование**: PHP CS Fixer с правилами `@PER-CS`
- **Отступы**: 4 пробела (согласно `.editorconfig`)
- **Конец строки**: LF
- **UTF-8**: кодировка файлов
- **Строгие типы**: `declare(strict_types=1)` в начале каждого PHP-файла

### Структура кода

1. **Имён классов**: PascalCase
2. **Методов и свойств**: camelCase
3. **Констант**: UPPER_SNAKE_CASE
4. **Файлов миграций**: `MYYMMDDHHMMName` (например, `M251102141707Page`)

### Тестирование

- **Фреймворк**: Codeception 5.x
- **Направления тестов**:
  - `tests/Unit/` — юнит-тесты
  - `tests/Functional/` — функциональные тесты
  - `tests/Web/` — веб-тесты
  - `tests/Console/` — тесты CLI-команд

### Статический анализ

- **Psalm**: уровень ошибок 1, кэш в `runtime/cache/psalm`
- **Rector**: поддержка PHP 8.2+ правил
- **Composer Dependency Analyser**: контроль используемых зависимостей

### Принципы разработки

1. **Один Use Case = один класс** — без бизнес-логики в контроллерах
2. **Интерфейсы репозиториев** — в domain/application, реализация — в infrastructure
3. **Агрегаты** — одна транзакционная граница = один агрегат
4. **Идемпотентность** — все внешние операции должны быть идемпотентными
5. **UUID первичные ключи** — генерация ID в коде, не в БД
6. **Неизменяемость (Immutability)** — Value Objects должны быть immutable

### Безопасность

- Валидация всех входных данных
- Защита от SQL injection (использовать query builder)
- CSRF-защита для форм
- RBAC для авторизации

### Git-практики

- Использовать `.gitignore` и `.gitattributes`
- Не коммитить `runtime/`, `vendor/`, `.env`
- Миграции БД должны быть в VCS

## Важные примечания

- Система работает с **несколькими источниками данных** (внешние API, внутренние операции)
- Источники могут **дублировать или противоречить** друг другу — система должна приводить данные к согласованному состоянию
- **Резервы** товаров уменьшают доступный остаток, снятие резерва — явная операция
- **Остатки не могут быть отрицательными**
- Изменения в `config/configuration.php` требуют запуска `composer yii-config-rebuild`
